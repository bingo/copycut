# Alpha / Step 2 范围定义（MVP Scope）

> 明确 Step 2 Presentation Mock 做什么、哪些真实能力推后到 Step 3，避免范围蔓延。

---

## Alpha 版本定义

- **形态**：Presentation 层 Mock（UI 完整，剪辑/AI 后端功能留空或用 stub）
- **目标**：完整的视觉交互原型，验证 UX 流程，可演示给早期用户
- **不包含**：真实视频渲染、AI 推理、云端存储
- **范围口径**：P0/P1/P2 表示产品优先级；实现阶段表示当前 Step 2 只做 Mock，还是 Step 3 接入真实能力

## 技术约束（Alpha 阶段）

| 约束 | 说明 |
|------|------|
| 视频渲染 | Alpha 阶段不做真实渲染，导出为 mock（显示进度后下载空文件或原始素材） |
| 本地存储 | Step 2 优先保存项目元数据；localStorage 可保存轻量配置，IndexedDB 可保存较大的 mock 数据 |
| 素材存储 | 默认不上传服务器；浏览器 File 对象不能稳定跨会话恢复，重新打开草稿时允许提示“请重新关联素材” |
| AI 功能 | 所有 AI 功能 UI 留空，按钮状态为"开发中" |
| 滤镜实现 | CSS filter 或 WebGL shader 实现实时预览，输出时可能降级为静态截图 |
| 文件格式 | Step 2 优先支持 MP4/MOV/JPG/PNG 的展示和预览；HEIC 可展示入口或提示转换，真实解码推后 |

## 评估标准

Alpha 版本通过以下场景即视为达标：

1. **核心流程可走通**：导入素材 → 剪辑 → 加滤镜 → 加文字 → 加音乐 → 制作封面 → 导出（mock）全流程无卡点
2. **UI 还原度**：80% 的功能有 UI 交互（即使后端留空）
3. **小红书用户可识别**：真实小红书创作者看到原型后能快速理解工作流，无需引导
4. **阶段边界清晰**：用户能理解 Alpha 导出、字幕、AI、真实渲染为演示能力，不误认为已完成后台处理
5. **发布准备有效**：用户能在导出前看到标题/正文/话题、封面安全区、音乐版权和水印风险提示
